# Runbook: Высокий уровень ошибок (High Error Rate)

## 1. Что означает этот алерт

Алерт срабатывает, когда QuickNotes возвращает ошибки (HTTP 5xx) на более чем 5% запросов в течение 5 минут. Это говорит о том, что сервис деградирует, и пользователи могут видеть ошибки.

---

## 2. Диагностика

### 2.1 Проверка доступа к сервису

```bash
curl -s http://localhost:8080/health
```

Ожидаемый ответ: `{"notes":X,"status":"ok"}`. Если ответа от сервиса нет, его необходимо перезапустить.

### 2.2 Просмотр логов

`docker compose logs quicknotes --tail=50 | grep -i error` - могут быть: panic, error, timeout.

### 2.3 Проверить заметки в Prometeus

`quicknotes\_notes\_total` - количество заметок, должно быть != 0.

## 3 Решение проблемы

### 3.1 Перезапуск сервиса

`docker compose restart quicknotes`

### 3.2 Откат к рабочей версии

```bash
docker compose down
docker compose up -d
```

## 4. После инцидента

1. Написать blameless postmortem;
2. Обновить этот runbook, если в процессе были выявлены новые методы устранения неполадок.
