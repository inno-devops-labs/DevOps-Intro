# Задание 1

## AWS CodeArtifact/ECR

Поддерживаемые форматы:
Docker (OCI), npm, PyPI, Maven, NuGet, Swift, RubyGems

Ключевые функции:
Сканирование образов (Clair/Amazon Inspector), репликация в другие регионы, интеграция с IAM

Интеграция:
Поддерживается с Lambda, ECS/EKS, CodeBuild, EventBridge

Оплата:
плата за объем хранения (ГБ) и количество запросов API

## GCP; Artifact registry

Поддерживаемые форматы:
Docker (OCI), npm, Maven, PyPI, NuGet, Go, Apt/Yum, Helm

Ключевые функции:
Сканирование на уязвимости, виртуальные и удаленные репозитории (прокси), управление жизненным циклом

Интеграция:
Поддерживается с GKE, Cloud Run, Cloud Build, Vertex AI

Оплата:
плата за хранение (в среднем 10 центов за ГБ после 500МБ, но в целом вроде от региона зависит) и исходящий трафик

## Azure Artifacts/ACR

Поддерживаемые форматы:
npm, NuGet, Maven, PyPI; Docker (OCI), Helm (ACR)

Ключевые функции:
Гео-репликация (в ACR Premium), доверенное подписание (Trust Signing), интеграция с AD

Интеграция:
Поддерживается с Azure DevOps (Pipelines), AKS, Azure Web Apps

Оплата:
плата за хранение (ГБ) плюс еще фиксированная плата за SKU (в случае Container Registry)

в ACR есть бесплатный лимит. Пока его не достиг можно кайфовать

## Сходства и различия
Все три провайдера поддерживают стандарты OCI. Все сервисы позволяют создавать закрытые хранилища с доступом только для авторизованных пользователей. Везде управление доступом завязано на внутренние системы прав (Identity and Access Management).

Различия: GCP — единственный, кто объединил хранение контейнеров и библиотек в одном интерфейсе. У AWS и Azure это разные сервисы. У ACR есть фиксированные тарифные планы (Basic/Standard/Premium) с разным набором функций, а другие требуют столько денег, сколько я пользовался сервисом.

# Задание 2

## AWS Lambda

Языки:
Node.js, Python, Java, Go, Ruby, .NET

Выполнение:
Event-driven, HTTP (через API Gateway)

Таймаут:
15 минут

Cold Start:
не высокий, средний; в общем не очень долго

Интеграция:
с S3, DynamoDB, SQS, Kinesis, EventBridge

Оплата:
за количество вызовов плюс ГБ/сек (время выполнения)

## Google Cloud Functions

Языки:
Node.js, Python, Go, Java, PHP, Ruby, .NET

Выполнение:
Event-driven, HTTP-triggered

Таймаут:
9-10 минут

Cold Start:
низкий, также зависит от языка программирования; быстрее чем AWS

Интеграция:
с Cloud Storage, Pub/Sub, Firestore, BigQuery

Оплата:
за количество вызовов плюс время работы процессора и памяти плюс за трафик

## Azure Functions

Языки:
.NET, Node.js, Python, Java, PowerShell, Go

Выполнение:
Event-driven, HTTP-triggered, Durable

Таймаут:
5–10 минут (если мажор, то 60 мин+ для премиум-пользователей)

Cold Start:
зависит от тарифа; опять-таки если мажор (премиум-пользователь), то вообще нет колд старта

Интеграция:
с Blob Storage, Cosmos DB, Service Bus, Event Grid

Оплата:
за количество вызовов плюс время работы процессора и памяти.

## Недостатки и преимущества Serverless

Преимущества: платишь только за то, чем пользуешься, то есть не как подписка. Не нужно ничего администрировать. Просто работаю, остальное сделают за меня.

Недостатки: наличие cold start. Ограниченность по времени непрерывной работы.
