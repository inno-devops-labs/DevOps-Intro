# Lab 9 — DevSecOps Tools

## Task 1 — Web Application Scanning with OWASP ZAP

Для сканирования использовалось намеренно уязвимое приложение OWASP Juice Shop,
запущенное локально в Docker на порту 3000. Сканирование выполнялось инструментом
ZAP версии 2.17.0 в режиме Baseline Scan. Всего было просканировано 123 URL.

### Результаты сканирования

Было найдено 2 уязвимости уровня Medium, 5 уровня Low и 3 информационных.

**Уязвимость 1 — Content Security Policy (CSP) Header Not Set**

Приложение не отправляет заголовок `Content-Security-Policy` в HTTP-ответах.
Это означает, что браузер не знает, каким источникам контента доверять,
что открывает возможность для XSS-атак — злоумышленник может внедрить
вредоносный скрипт, который выполнится в браузере жертвы. Затронуто 11 URL,
включая главную страницу.

**Уязвимость 2 — Cross-Domain Misconfiguration**

Приложение использует слишком широкие CORS-настройки — фактически разрешает
запросы с любого домена (`Access-Control-Allow-Origin: *`). Это позволяет
сторонним сайтам делать запросы от имени пользователя и потенциально
получать его данные. Тоже затронуто 11 URL.

### Security Headers

Заголовок `Content-Security-Policy` отсутствует — без него невозможно
ограничить источники скриптов и предотвратить XSS. Заголовок
`Cross-Origin-Embedder-Policy` тоже не установлен (12 страниц), что
позволяет встраивать ресурсы с других доменов без ограничений.
При этом заголовки `Strict-Transport-Security`, `X-Content-Type-Options`
и защита от кликджекинга присутствуют — это плюс.
Также замечен устаревший заголовок `Feature-Policy`, который давно
заменён на `Permissions-Policy`.

### Анализ

Самые распространённые уязвимости в веб-приложениях — это именно
отсутствие security headers и неправильная настройка CORS. Причина
простая: фреймворки по умолчанию не добавляют эти заголовки, а
разработчики о них не думают, пока не приходит пентестер или сканер.
XSS и инъекции тоже встречаются постоянно, потому что входные данные
от пользователя слишком часто попадают в HTML или SQL без валидации.


## Task 2 — Container Vulnerability Scanning with Trivy

Для сканирования использовался образ `bkimminich/juice-shop` с инструментом
Trivy версии 0.61.0. ОС внутри образа — Debian 13.4.

### Результаты сканирования

Всего найдено **41 уязвимость HIGH и 10 CRITICAL** в Node.js-пакетах.
Дополнительно обнаружен 1 HIGH-секрет — приватный ключ прямо в исходном коде
(`juice-shop/build/lib/insecurity.js`).

### Уязвимые пакеты

**jsonwebtoken — CVE-2015-9235 (CRITICAL)**  
Версия 0.1.0. Уязвимость позволяет полностью обойти проверку подписи JWT-токена
с помощью подменённого токена. Это означает, что злоумышленник может сгенерировать
поддельный токен и авторизоваться под любым пользователем, включая администратора.
Исправлено в версии 4.2.2.

**vm2 — CVE-2023-32314 (CRITICAL)**  
Версия 3.9.17. Побег из песочницы (Sandbox Escape) — атакующий может выполнить
произвольный код за пределами изолированного окружения vm2. Это особенно опасно,
потому что vm2 используется для запуска ненадёжного кода в изоляции, и если
изоляция ломается, под угрозой оказывается вся система. Исправлено в 3.9.18.

**marsdb — GHSA-5mrr-rgp6-x4gr (CRITICAL)**  
Версия 0.6.11. Command Injection — внедрение произвольных команд операционной
системы через входные данные. Патча нет совсем, пакет заброшен.

**node-tar — множество CVE (HIGH)**  
Несколько уязвимостей с path traversal — злоумышленник через специально
созданный архив может перезаписывать или читать произвольные файлы на хосте.

### Наиболее частый тип уязвимостей

Самый распространённый тип — это **path traversal и arbitrary file
operations** (в пакете `node-tar` сразу несколько CVE), а также
**произвольное выполнение кода** (vm2, handlebars, marsdb).

### Почему важно сканировать образы перед деплоем

Даже официальные образы из Docker Hub содержат устаревшие системные библиотеки
и npm-пакеты с известными CVE. В данном случае в образе нашлось 10 критических
уязвимостей, включая полный обход аутентификации JWT и побег из песочницы.
Если задеплоить такой образ в продакшн, атакующему достаточно одного запроса,
чтобы получить права администратора или выполнить произвольный код на сервере.

### Интеграция в CI/CD

В GitHub Actions это выглядело бы так:

```yaml
- name: Scan image with Trivy
  uses: aquasecurity/trivy-action@master
  with:
    image-ref: myapp:latest
    severity: CRITICAL,HIGH
    exit-code: 1  # фейлит билд при наличии критических уязвимостей
```

Флаг `exit-code: 1` автоматически останавливает пайплайн, если найдены
CRITICAL-уязвимости — это и есть принцип "shift left": ловить проблемы
безопасности на этапе сборки, а не после деплоя.
