- Анализ

Были выбраны API check и Browser check, так как они проверяют разные уровни работы сайта.
API check проверяет доступность сервера и корректный ответ (status 200), что позволяет быстро выявить проблемы на уровне бэкенда.
Browser check имитирует пользователя и проверяет, что страница реально загружается.

Пороговые значения выбраны простые: регулярные проверки и оповещение при первом сбое, чтобы быстро обнаруживать проблемы без лишней сложности.

- Размышления

Такая система мониторинга помогает быстро выявлять сбои и снижать время простоя сайта.
API check показывает техническую доступность, а Browser check — корректность работы сайта для пользователя.

Уведомления позволяют оперативно реагировать на ошибки, что повышает надежность системы.
