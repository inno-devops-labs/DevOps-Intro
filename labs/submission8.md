## 5) Analysis: Why did you choose these specific checks and thresholds?

I chose example.com because it is a stable and publicly available website suitable for monitoring.

The API check verifies basic availability of the website by ensuring that the server responds with HTTP status code 200. This helps confirm that the backend is reachable.

The browser check validates that the page loads correctly and that the expected content ("Example Domain") is visible. This simulates a real user experience and ensures that not only the server is up, but the content is rendered properly.

The alert is configured to trigger immediately when a check fails. This allows fast detection of issues such as downtime or broken content.

---

## 6) Reflection: How does this monitoring setup help maintain website reliability?

This monitoring setup provides both technical and user-level validation.

API checks ensure that the system is available at the network level, while browser checks confirm that the website works correctly from a user perspective.

Alerts enable quick reaction to failures, reducing downtime and helping maintain high reliability and availability of the service.
