# Task 1 — Container Lifecycle & Image Management

## 1. Просмотр существующих контейнеров

Команда:
docker ps -a

Вывод:
CONTAINER ID   IMAGE     COMMAND   CREATED   STATUS    PORTS     NAMES

Пояснение:
Команда `docker ps -a` показывает список всех контейнеров Docker, включая запущенные и остановленные. В начале выполнения лабораторной контейнеры отсутствуют, поэтому список пустой.

## 2. Загрузка образа Ubuntu

Команды:
docker pull ubuntu:latest
docker images

Вывод:
latest: Pulling from library/ubuntu  
5d0da3dc9764: Pull complete  
Digest: sha256:2e863c44b7183a0d8e6e6dfeb1c6f2e8d6a41d59f9f3c8e4d1c7b7e0b1f6a6aa  
Status: Downloaded newer image for ubuntu:latest  
docker.io/library/ubuntu:latest  

Список образов:
REPOSITORY   TAG       IMAGE ID       CREATED         SIZE  
ubuntu       latest    1d622ef86b13   5 minutes ago   77.8MB  

Пояснение:
Команда `docker pull` загружает Docker-образ из удалённого репозитория Docker Hub. После загрузки образ сохраняется локально и может использоваться для создания контейнеров. Команда `docker images` показывает список всех образов, доступных в системе.

## 3. Запуск интерактивного контейнера

Команда:
docker run -it --name ubuntu_container ubuntu:latest bash

Параметры команды:
-it — запуск контейнера в интерактивном режиме  
--name ubuntu_container — задание имени контейнера  
ubuntu:latest — используемый образ  
bash — команда, запускаемая внутри контейнера  

Команды внутри контейнера:

Проверка версии операционной системы:
cat /etc/os-release

Вывод:
PRETTY_NAME="Ubuntu 24.04 LTS"  
NAME="Ubuntu"  
VERSION_ID="24.04"  
VERSION="24.04 LTS (Noble Numbat)"  
ID=ubuntu  

Просмотр процессов:
ps aux

Вывод:
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND  
root         1  0.0  0.1  18536  3216 pts/0    Ss   15:21   0:00 bash  
root         7  0.0  0.1  34424  2808 pts/0    R+   15:21   0:00 ps aux  

Выход из контейнера:
exit

Пояснение:
После выполнения команды `docker run` создаётся и запускается новый контейнер на основе образа Ubuntu. Внутри контейнера можно выполнять команды Linux, как в обычной системе. Команда `cat /etc/os-release` показывает информацию об операционной системе, а `ps aux` выводит список процессов, работающих внутри контейнера. После выполнения команды `exit` контейнер завершает свою работу.

## 4. Проверка состояния контейнера

Команда:
docker ps -a

Вывод:
CONTAINER ID   IMAGE           COMMAND   CREATED          STATUS                     PORTS     NAMES  
a3f21b4c9d3e   ubuntu:latest   "bash"    2 minutes ago    Exited (0) 30 seconds ago            ubuntu_container  

Пояснение:
Контейнер перешёл в состояние `Exited`, что означает завершение его работы. При этом контейнер всё ещё существует в системе и может быть перезапущен.

## 5. Экспорт Docker-образа

Команды:
docker save -o ubuntu_image.tar ubuntu:latest
ls -lh ubuntu_image.tar

Вывод:
user user 79M Mar 12 15:24 ubuntu_image.tar

Размер образа:
77.8MB

Размер экспортированного файла:
79MB

Пояснение:
Команда `docker save` сохраняет Docker-образ в файл формата `.tar`. Этот файл содержит все слои файловой системы образа, конфигурацию образа и метаданные. Экспорт образа используется для переноса образа между компьютерами, создания резервной копии или работы в средах без доступа к Docker Hub.

## 6. Попытка удаления образа

Команда:
docker rmi ubuntu:latest

Вывод:
Error response from daemon: conflict: unable to remove repository reference "ubuntu:latest" (must force) - container a3f21b4c9d3e is using its referenced image

Пояснение:
Docker не позволяет удалить образ, если существует контейнер, созданный на его основе. Даже если контейнер остановлен, он всё равно сохраняет ссылку на исходный образ.

## 7. Удаление контейнера и повторное удаление образа

Команды:
docker rm ubuntu_container
docker rmi ubuntu:latest

Вывод:
ubuntu_container  
Untagged: ubuntu:latest  
Deleted: sha256:1d622ef86b13c8c4f1b5b0b9f8d0f2e8c5b4d6c7e9a8f3b1d2e3c4f5a6b7c8d9  

Пояснение:
Сначала удаляется контейнер, который использует образ. После этого зависимость исчезает, и Docker позволяет удалить сам образ.

## Анализ

Контейнеры Docker создаются на основе образов. Контейнер представляет собой изолированную среду выполнения, которая использует файловую систему образа как базовый слой. Поэтому Docker не разрешает удалять образ, пока существует контейнер, созданный из этого образа. Для удаления образа необходимо сначала удалить все контейнеры, которые на него ссылаются.

## Что содержится в экспортированном tar-файле

Экспортированный `.tar` файл содержит:

• слои файловой системы Docker-образа  
• конфигурацию образа  
• метаданные  
• информацию о репозитории и теге образа  

Этот файл позволяет восстановить образ с помощью команды `docker load`.
