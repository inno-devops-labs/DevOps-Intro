# Изученые ключевые концепции:
# Workflow — YAML-файл в .github/workflows/ описывает автоматизацию
# Job — набор шагов, выполняемых на одном runner
# Step — отдельная команда или действие внутри job
# Runner — виртуальная машина (ubuntu-latest), где выполняется job
# Trigger — событие, запускающее workflow (в данном случае push)

# После выполнения push GitHub обнаружил изменение и запустил workflow. Был выделен runner ubuntu-latest. Сначала репозиторий был скачан с помощью actions/checkout, затем выполнены команды для вывода информации. Job завершился успешно (Success)

# В файл .github/workflows/main.yml был добавлен ручной триггер workflow_dispatch. Также был добавлен дополнительный шаг для сбора системной информации о runner

# В ходе выполнения workflow были получены следующие данные:
# Операционная система: Ubuntu (ubuntu-latest)
# Информация о ядре: вывод команды uname -a
# Характеристики CPU: вывод lscpu
# Объём оперативной памяти: вывод free -h
# Информация о диске: вывод df -h

# Автоматический запуск (push) происходит при отправке изменений в репозиторий. 
# Ручной запуск (workflow_dispatch) выполняется вручную через кнопку Run workflow в интерфейсе GitHub. Ручной запуск удобен для тестирования без необходимости делать новый commit.

# Workflow выполняется на GitHub-hosted runner с операционной системой Ubuntu. Runner предоставляет стандартную Linux-среду с предустановленными инструментами (git, python, node и др.). Среда подходит для автоматической сборки, тестирования и выполнения скриптов
